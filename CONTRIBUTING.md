# Contributing to Pump Up

First off — thank you. The fact that you want to help makes our day. 🙏

There's one issue worth knowing up front, and then lots of ways to get involved.

## This repo is a mirror

All of Pump Up's code is authored in an upstream monorepo and **mirrored out to this repo automatically** on every change. That keeps the SDKs, integrations, and docs in lockstep with the backend they talk to.

The practical upshot: this repo's `main` is overwritten by the sync on every push, so **pull requests are turned off here** — there's nowhere for them to land. No hard feelings — the best ways to help are right below. 💙

## How to actually help

These are the things that genuinely move the product forward, and we read every one:

- **🐛 File an issue.** Hit a bug, a confusing error, a docs gap, a rough edge in an SDK or integration? [Open an issue](https://github.com/pumpupai/pumpup/issues). Concrete repros are gold.
- **💡 Start a discussion.** Want a feature, a new framework integration, or just want to talk through how you'd use Pump Up? [Discussions](https://github.com/pumpupai/pumpup/discussions) is the place. Telling us *which* integration you want next directly shapes what we build.
- **⭐ Star the repo.** It's a small thing, but it genuinely helps other developers find us.
- **🔒 Report security issues privately.** Please don't use public issues for vulnerabilities — see [SECURITY.md](./SECURITY.md).

## Found something broken in the code?

Even though we can't take a PR fix, please **still tell us** — open an issue describing what's wrong (and where, if you spotted it). We'll fix it upstream and the change will flow back here on the next sync, usually fast.

Thanks for being here. We're building this in the open because we want to build it *with* you. 🚀
